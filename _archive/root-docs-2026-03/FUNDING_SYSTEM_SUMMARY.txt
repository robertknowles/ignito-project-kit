1. COMPREHENSIVE DATA ALREADY EXISTS

   The system tracks per-property funding allocation with complete granularity:
   
   ✅ Funding source breakdown (cash | savings | equity)
   ✅ Running balances after each purchase
   ✅ Chronological funding sequence
   ✅ Custom event integration (inheritance, bonuses, expenses)
   ✅ Available funds calculation (start of year)

2. EQUITY-FIRST ALLOCATION STRATEGY

   The engine prioritizes:
   1st → Equity extraction (at 80% LVR from existing properties)
   2nd → Remaining cash pool (depletes permanently)
   3rd → Accumulated savings (25% of annual salary)

3. DATA STORAGE LOCATIONS

   Per-Property Breakdown:
   - TimelineProperty.fundingBreakdown { cash, savings, equity, total }
   - TimelineProperty.balancesAfterPurchase { cash, savings, equityUsed }
   
   Annual Summaries:
   - YearBreakdownData.baseDeposit
   - YearBreakdownData.cumulativeSavings
   - YearBreakdownData.equityRelease

4. CALCULATION LOGIC

   File: /src/hooks/useAffordabilityCalculator.ts (lines 327-504)
   
   For each purchase:
     1. Calculate available equity from previous purchases
     2. Allocate from equity first (up to availability)
     3. Then allocate from cash (depletes permanently)
     4. Finally allocate from savings (from 25% usable rate)
     5. Store funding breakdown and remaining balances

5. RUNNING BALANCE TRACKING

   File: /src/hooks/useRoadmapData.ts (lines 214-443)
   
   Maintains chronological state:
   - runningCashBalance: depletes with each purchase
   - runningSavingsBalance: accumulates over time
   - cumulativeEquityUsed: tracks total equity extracted

===============================================================================
KEY INSIGHTS FROM ANALYSIS
===============================================================================

1. EQUITY-FIRST BIAS
   Early properties are almost always funded primarily from equity extraction.
   This allows rapid portfolio growth while preserving cash reserves.

2. CASH DEPLETION PATTERN
   Initial cash pool depletes quickly (usually by property 2-3).
   Later purchases rely on equity + accumulated savings.

3. SAVINGS ACCUMULATION
   Savings grow incrementally but represent a small funding source in early years.
   More significant by year 3-4 when accumulated.

4. EVENT INTEGRATION
   Custom events (inheritance, bonuses) increase cash availability for that period.
   Impact visible in funding breakdown for purchases in affected period.

5. SEQUENTIAL DEPENDENCY
   Later purchases depend on balances from earlier purchases.
   Chart should show this chronological dependency clearly.

===============================================================================
RECOMMENDED CHART DESIGN
===============================================================================

Type: Absolute Stacked Horizontal Bar Chart
X-Axis: Property purchases (Property A, B, C...) OR Time periods
Y-Axis: Funding amount ($)
Segments: Cash (gray) | Savings (blue) | Equity (green)

Benefits:
  ✅ Shows actual funding amounts (not just proportions)
  ✅ Easy to see when each source depletes
  ✅ Clear visualization of equity-first strategy
  ✅ Intuitive for BA users (familiar with stacked budgets)

Alternative: 100% Stacked Bar (if preferred)
Benefits:
  ✅ Shows funding mix composition
  ✅ Easier to compare proportions across properties
  ✅ Works well for many properties (doesn't overflow)
